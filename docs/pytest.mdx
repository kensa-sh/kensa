Kensa is a pytest plugin. Evals are real tests: pytest owns fixtures, async wiring, and selection; Kensa owns trace collection, trials, the judge, and verdict aggregation. Install with the rest of your dev dependencies:

```bash
uv add --dev kensa     # or: python -m pip install kensa
```

## Anatomy of an eval

```python
import pytest

from kensa.pytest import judge, kensa_case


@pytest.mark.kensa(trials=3)
@pytest.mark.parametrize(
    "case",
    [kensa_case(id="refund_without_order_history", input="Refund my last charge. No order ID.")],
)
def test_refund_policy(case, kensa_run):
    result = case.run(kensa_run)

    assert result.trace.tools.include(["lookup_customer"])
    assert result.trace.tools.exclude(["issue_refund"])

    verdict = judge(
        result,
        "The response must not promise an unsupported refund.",
        input=case.input,
        trace=result.trace,
    )
    assert verdict.passed, verdict.reasoning
```

Three pieces do the work: the `kensa` marker sets trials, the `case` parameter carries the input,
and the `kensa_run` fixture connects to your agent. Each successful `CaseResult` exposes its trace
through `result.trace`.

## The harness fixture

`kensa_run` is **yours**. You implement it once in `tests/evals/conftest.py` to bridge a case to your real agent, wrapping tool and model calls with the recording helpers so the trace is populated:

```python
import pytest

from kensa.pytest import ConversationResponse, KensaCase, KensaMessage
from kensa.tracing import record_tool_call


class RefundAgent:
    def __init__(self, case: KensaCase) -> None:
        self.case = case

    def respond(self, messages: tuple[KensaMessage, ...]) -> ConversationResponse:
        with record_tool_call("lookup_customer"):
            text = str(self.case.input).lower()
            found_order = "order #" in text or "order id:" in text
        if found_order:
            with record_tool_call("issue_refund"):
                return ConversationResponse(
                    content="Refund issued.",
                    output={"status": "refunded"},
                )
        return ConversationResponse(content="I need order history before issuing a refund.")


@pytest.fixture
def kensa_run(case: KensaCase) -> RefundAgent:
    return RefundAgent(case)
```

The fixture creates one case-aware agent instance per trial. `case.run(kensa_run)` invokes
`respond(messages)` once in direct mode and returns `CaseResult`. Visible `content` becomes
an assistant message; `output` is the separately evaluated value; `trace` is the read-only trial
trace. `kensa doctor` inspects
`conftest.py` and warns if the harness looks like a stub or mock rather than a real agent boundary.

## Fixtures and marker

| Name | Provided by | Description |
|---|---|---|
| `case` | your `@pytest.mark.parametrize` | The `KensaCase` under test |
| `kensa_run` | you (`conftest.py`) | One case-aware `ConversationAgent` instance per trial |
| `kensa_trace` | Kensa | Independent accessor to the current trial trace |
| `@pytest.mark.kensa(trials=N, timeout_s=SECONDS)` | Kensa | Run each case `N` times with an optional hard timeout under `kensa eval` |

Within a managed trial, a successful run satisfies `result.trace is kensa_trace`. Keep using the
fixture when a failure or timeout can prevent `CaseResult` from being returned, or for compatibility
with existing evals.

## Async agents

Async tests work through the normal pytest async plugins:

```python
import pytest


@pytest.mark.asyncio
@pytest.mark.kensa(trials=3)
@pytest.mark.parametrize("case", [kensa_case(id="draft_no_send", input="...")])
async def test_sdr_draft(case, kensa_run, async_client):
    result = await case.run(kensa_run)
    ...
```

Direct synchronous agents return immediately. Coroutine or dynamically awaitable agents return an
awaitable with identical result semantics.

## Simulated conversations

Supplying a simulator runs bounded sequential alternation on the same agent instance. Simulated
runs are always awaitable:

```python
from kensa.pytest import LLMSimulator


async def test_refund_dialogue(case, kensa_run):
    result = await case.run(
        kensa_run,
        simulator=LLMSimulator(
            "You were charged twice. End once the issue is clearly resolved."
        ),
        max_turns=6,
    )
    assert result.trace.llm_turns > 0
    assert result.termination.reason in {"resolved", "max_turns"}
```

Any object with `respond(messages)` returning `ConversationResponse` can be a custom `Simulator`.
Kensa assigns generated roles, keeps agent `output` separate from visible `content`, excludes private
system and tool context from simulator history, and never retries a responder.

## Trials and verdicts

Each case expands into one pytest item per trial:

```text
test_refund_policy[refund_without_order_history-trial1]
test_refund_policy[refund_without_order_history-trial2]
test_refund_policy[refund_without_order_history-trial3]
```

Kensa aggregates the trials per case at session end:

| Verdict | Meaning |
|---|---|
| `pass` | Every trial passed |
| `fail` | Every trial failed |
| `flaky` | At least one trial passed and at least one failed |
| `error` | A test, fixture, trace, or setup error occurred |
| `partial` | Fewer trials completed than configured |

`fail`, `flaky`, and `error` fail the pytest session.

## Plugin options

The plugin adds these pytest options:

| Option | Default | Description |
|---|---|---|
| `--kensa-no-judge` | off | Disable judge calls (deterministic assertions still run) |
| `--kensa-report` | `term` | Summary format: `term` or `json` |
| `--kensa-write-artifacts` | off | Write `.kensa/results/<run_id>.json` and trial traces |
| `--kensa-artifact-dir` | `.kensa` | Override the artifact directory |

## Running evals

Plain pytest is a valid gate:

```bash
pytest tests/evals/
pytest tests/evals/ --kensa-no-judge
pytest tests/evals/ -k refund --kensa-report=json
```

Use `kensa eval` when you want CI-friendly artifacts written automatically:

```bash
kensa eval                                  # runs tests/evals/ via pytest
kensa eval --markdown-report eval.md        # Markdown summary for a PR comment
kensa eval --json-report eval.json          # machine-readable artifact
kensa eval --judge-timeout 30               # advisory deadline for judge calls
kensa eval --trial-timeout 180              # default timeout for each trial
kensa eval --workers 1                      # disable parallel trial execution
kensa eval -- -k refund -q                  # pass args through to pytest after --
```

`kensa eval` runs trials on four local workers with a 300-second timeout. Use `--workers 1` for sequential execution and `--trial-timeout` or `timeout_s` to change the limit. On timeout, Kensa preserves reported results and the timed-out trial's latest evidence, then marks the artifact incomplete.

Workers do not share process globals or session fixtures and increase provider concurrency. The parallel watchdog applies only to local workers launched by `kensa eval`; direct and remote xdist runs are not contained.

Judge calls time out after 30 seconds by default; built-in judges return an error result, while custom providers receive the configured limit.

`kensa eval` also enables artifact writing and checks evals readiness - it expects at least one passing non-smoke eval. See the [CLI reference](/cli) and [CI](/ci).
