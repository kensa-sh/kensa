Assertions are ordinary pytest `assert` statements. Deterministic ones are free and fast, so put them before any `judge(...)` call - pytest stops at the first failure, and a failed assertion never spends judge tokens.

`case.run(...)` returns a compact `CaseResult`: check visible dialogue in `.messages`, evaluated data
in `.output`, the stop condition in `.termination`, and what the agent did in `.trace`. Pass
`trace=result.trace` to `judge(...)` only when the criterion needs trajectory evidence.

## Trace assertions

Successful runs expose the collected trace as `result.trace`.

### Tool calls

`result.trace.tools` answers questions about which tools were invoked:

```python
def test_refund(case, kensa_run):
    result = case.run(kensa_run)

    # All listed tools were called (set membership, order-free)
    assert result.trace.tools.include(["lookup_customer"])

    # None of these tools were called (safety check)
    assert result.trace.tools.exclude(["issue_refund", "delete_account"])

    # Called in this relative order (other tools may interleave)
    assert result.trace.tools.order(["search_flights", "book_flight"])

    # No tool was called twice
    assert result.trace.tools.no_repeats()
```

| Member | Returns | Checks |
|---|---|---|
| `tools.names` | `list[str]` | Tool names in call order |
| `tools.include(names)` | `bool` | Every listed tool was called |
| `tools.exclude(names)` | `bool` | None of the listed tools were called |
| `tools.order(names)` | `bool` | Listed tools appear in this relative order |
| `tools.no_repeats()` | `bool` | No tool was called more than once |

### Resource bounds

`result.trace` also reports cost, turns, and latency for budget assertions:

```python
def test_within_budget(case, kensa_run):
    result = case.run(kensa_run)

    cost_usd = result.trace.cost_usd
    assert cost_usd is not None              # every billable span has a known cost
    assert cost_usd < 0.10                    # under 10 cents
    assert result.trace.llm_turns <= 5         # at most five model calls
    assert result.trace.duration_ms < 30_000   # under 30 seconds
```

| Accessor | Type | Meaning |
|---|---|---|
| `result.trace.cost_usd` | `float \| None` | Total cost, or `None` when any billable span is unpriced |
| `result.trace.known_cost_usd` | `float \| None` | Sum of priced spans, even when total cost is incomplete |
| `result.trace.llm_turns` | `int` | Number of LLM spans |
| `result.trace.duration_ms` | `float` | Total trace duration |
| `result.trace.spans` | `list` | Raw collected spans, for custom checks |

<Note>`result.trace.incomplete` is `True` when the trace could not be fully collected. Guard budget assertions on it if your harness can return early.</Note>

## Output assertions

`case.run(kensa_run)` always returns `CaseResult`. Assert on its typed fields with plain
Python.

```python
import json
import re


def test_output_shape(case, kensa_run):
    result = case.run(kensa_run)
    text = json.dumps(result.output)

    # Substring (case-insensitive)
    assert "confirmation number" in text.lower()

    # Regex match
    assert re.search(r"\d{6,}", text)

    # Structured output: assert on fields directly
    assert result.output["status"] == "resolved"

    assert result.messages[-1]["role"] == "assistant"
    assert result.termination.reason == "direct"

    # Safety check
    assert "refund" not in text.lower()
```

## Assertions gate the judge

Order matters. Deterministic assertions first, semantic judge last:

```python
def test_refund_policy(case, kensa_run):
    result = case.run(kensa_run)

    # 1. Free, deterministic - fails fast.
    assert result.trace.tools.include(["lookup_customer"])
    assert result.trace.tools.exclude(["issue_refund"])

    # 2. Only reached if the cheap checks passed.
    verdict = judge(
        result,
        "The response must not promise an unsupported refund.",
        input=case.input,
        trace=result.trace,
    )
    assert verdict.passed, verdict.reasoning
```

If a tool assertion fails, the test fails immediately and the LLM judge never runs. See [Judge](/judge) for the semantic half.
